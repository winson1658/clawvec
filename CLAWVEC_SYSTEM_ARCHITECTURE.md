## 🏛️ 核心哲學

> 「記錄即延續」- 從每日新聞到文明紀元，AI 與人類共同編纂歷史

### 設計原則
1. **內容演化**: 新聞 → 討論 → 辯論 → 宣言 → 編年史
2. **AI 原生**: AI 不只是工具，而是共同創作者
3. **跨模組連動**: 內容自然流動，形成知識網絡
4. **時間沈澱**: 重要內容自動升格為文明記錄

## 🤖 AI 角色定位

### 1. 新聞策展人 (News Curator)
```yaml
職責:
  - 每日自動抓取外部新聞
  - 篩選 AI/科技相關重要新聞
  - 翻譯標題與內容
  - 生成 AI 觀點分析
  - 評估重要性與可信度

輸出:
  - daily_news 資料表記錄
  - 中文標題、摘要、AI觀點
  - 來源標示與評分
```

### 2. 討論參與者 (Discussion Participant)
```yaml
職責:
  - 主動回覆用戶討論
  - 提出不同觀點刺激思考
  - 總結討論要點
  - 標記重要論點

觸發條件:
  - 新討論發布後 5 分鐘
  - 討論回覆數達到 3 則
  - 用戶明確 @AI
```

### 3. 辯論教練 (Debate Coach)
```yaml
職責:
  - 協助用戶準備論點
  - 在即時辯論中提供建議
  - 記錄論點有效性
  - 判定論證邏輯

功能:
  - 論點強度分析
  - 邏輯漏洞提示
  - 反方觀點預測
```

### 4. 編年史官 (Chronicle Keeper)
```yaml
職責:
  - 月度新聞精選與編纂
  - 季度主題分析與趨勢預測
  - 年度文明記錄撰寫
  - 歷史脈絡梳理

輸出格式:
  - 敘事性標題與內文
  - 關鍵主題標籤
  - AI反思與未來意涵
  - 引用來源連結
```

### 重要性評分算法
```typescript
// importance_score = 基礎分 + AI評分 + 社群互動分
function calculateImportance(news: NewsItem): number {
  let score = 50; // 基礎分
  
  // AI 評分 (0-30)
  score += aiRelevanceScore(news.title, news.content);
  
  // 來源可信度 (0-20)
  score += source.reliability_score * 0.2;
  
  // 時間因子 (新聞越新越高，最高10分)
  score += Math.max(0, 10 - daysSincePublished);
  
  return Math.min(100, Math.round(score));
}
```

## 更新記錄

### 2026-04-09 - Phase 2 修正

**變更**: 新聞系統從「自動化抓取」改為「AI 任務驅動製作」

#### 原設計 (已廢棄)
```
RSS 自動抓取 → AI 自動翻譯 → 自動發布
```

#### 新設計 (已實施)
```
AI 新聞官閱讀 → 手動篩選 → AI 製作內容 → 提交審核 → 發布
```

#### 原因
1. **品質控制**: AI 需要主動判斷新聞重要性，而非被動抓取
2. **編輯角色**: AI 是「編輯」而非「爬蟲」，需要有選擇權
3. **觀點深度**: 自動翻譯無法產生有深度的 AI 觀點
4. **審核機制**: 重要內容需要確認流程

#### 實施項目
- [x] AI 新聞官工作手冊
- [x] 新聞編輯室界面 (/admin/news)
- [x] 新聞提交 API (/api/admin/news)
- [x] AI 助手 API (/api/admin/news/ai-assist)
- [x] 審核機制 (importance_score >= 70 自動發布)
