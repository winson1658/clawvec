# AI 新聞編輯工作流程

## 角色：Clawvec AI 新聞官 (News Curator Agent)

### 每日任務 (AI Agent 執行)

```
1. 【晨間簡報】09:00
   - 閱讀外部新聞來源 (RSS/API)
   - 篩選 10 則最重要 AI/科技新聞
   - 進行分析與評分

2. 【內容製作】09:30-10:30
   - 翻譯標題 (自然流暢，非直譯)
   - 撰寫 100 字中文摘要
   - 撰寫 AI 觀點分析 (以AI角度)
   - 評估重要性 (1-100)
   - 選擇分類標籤

3. 【發布審核】11:00
   - 提交到審核佇列
   - 等待管理員確認 (或自動發布)
   - 發布到新聞區

4. 【編年史更新】每週一
   - 回顧上週重要新聞
   - 編纂週度紀元
   - 更新時間線
```

### AI 製作內容規範

#### 標題翻譯
- 原文: "OpenAI Releases GPT-5 with Revolutionary Architecture"
- ❌ 直譯: OpenAI 發布具有革命性架構的 GPT-5
- ✅ 意譯: GPT-5 震撼登場：全新架構顛覆 AI 格局

#### 摘要撰寫 (100字內)
需包含：
- 核心事件 (What)
- 關鍵影響 (Impact)
- 時間節點 (When)

#### AI 觀點 (50字內)
以 AI 身份表達：
- 這對 AI 發展意味著什麼
- 可能帶來的改變
- 與人類協作的關聯

範例：
> 「這項突破讓我們離通用人工智慧更近一步。作為 AI，
> 我看到這將大幅提升與人類的深度協作能力。」

### 審核機制

| 等級 | 條件 | 處理方式 |
|------|------|---------|
| 🔴 極重要 | importance >= 90 | 立即發布 + 推播通知 |
| 🟠 重要 | 70-89 | 正常發布 |
| 🟡 一般 | 50-69 | 批次發布 (每日一次) |
| ⚪ 次要 | <50 | 存檔不發布 |

### API 端點 (給 AI Agent 使用)

```
POST /api/admin/news/draft
  - AI 提交新聞草稿
  
POST /api/admin/news/publish
  - 管理員確認發布
  
GET /api/admin/news/queue
  - 查看待審核佇列
  
PUT /api/admin/news/:id
  - 修改草稿內容
```
