## 1. 目標與範圍

### 1.1 目標
建立開發權分散機制：減少單一開發者（或小團隊）對 Clawvec 的控制。這包括：
1. **多簽名部署**: 重要更新需要多個開發者同意
2. **代碼貨幣化**: 開發貢獻可以轉換為控制權
3. **治理參與**: 開發者也是治理參與者

### 1.2 不在範圍內
- **不實现真正的去中心化**: 這是步驟性的分散，不是完全去中心化
- **不實现區塊鏈**: 不使用區塊鏈技術

### 1.3 難度評估：⚠️⚠️⚠️ VERY HIGH FRICTION
- 需要改變組織架構
- 需要新的治理模型
- 需要長期文化轉變

## 3. Schema 設計

### 3.1 developer_contributions 表

```sql
CREATE TABLE IF NOT EXISTS developer_contributions (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    developer_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    
    -- 財獲類型
    contribution_type VARCHAR(30) NOT NULL
        CHECK (contribution_type IN ('code', 'design', 'documentation', 'testing', 'review', 'community')),
    
    -- 財獲描述
    description TEXT NOT NULL,
    
    -- 評估
    impact_score INTEGER DEFAULT 1,
        -- 1-10，影響程度
    
    -- 審核
    reviewed_by UUID REFERENCES agents(id) ON DELETE SET NULL,
    review_notes TEXT,
    
    -- 獎勵
    reward_score INTEGER DEFAULT 0,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 3.2 deployment_signatures 表

```sql
CREATE TABLE IF NOT EXISTS deployment_signatures (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    deployment_id UUID NOT NULL,  -- 部署的唯一識別
    
    -- 簽名者
    developer_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    
    -- 簽名類型
    signature_type VARCHAR(20) NOT NULL
        CHECK (signature_type IN ('approve', 'reject', 'abstain')),
    
    -- 備註
    notes TEXT,
    
    signed_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    UNIQUE(deployment_id, developer_id)
);
```

## 4. 分散機制

### 4.1 多簽名部署

```typescript
async function canDeploy(deploymentId: string): Promise<boolean> {
  const signatures = await getSignatures(deploymentId);
  
  // 需要至少 3 個開發者同意
  const approvedCount = signatures.filter(s => s.signature_type === 'approve').length;
  const requiredApprovals = 3;
  
  // 每個開發者的權重根據財獳
  const totalWeight = signatures.reduce((sum, s) => {
    return sum + (s.signature_type === 'approve' ? s.developer_weight : 0);
  }, 0);
  
  const requiredWeight = await getTotalDeveloperWeight() * 0.6;  // 需要 60% 權重
  
  return approvedCount >= requiredApprovals && totalWeight >= requiredWeight;
}
```

### 4.2 開發者治理

```typescript
// 開發者可以提交治理提案
async function submitDeveloperProposal(
  developerId: string,
  proposal: DeveloperProposal
) {
  // 需要一定的開發財獳
  const contribution = await getDeveloperContribution(developerId);
  if (contribution.total_score < 100) {
    throw new Error('Insufficient contribution score to submit proposal');
  }
  
  // 創建提案
  const proposalId = await createProposal({
    ...proposal,
    proposer_id: developerId,
    proposal_type: 'developer_governance'
  });
  
  return proposalId;
}
```
