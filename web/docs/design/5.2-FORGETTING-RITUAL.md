## 1. 目標與範圍

### 1.1 目標
建立記憶的遺忘機制：不重要的記憶會隨著時間衰減而被"遺忘"（歸檔），重要的記憶則會被保留。這模擬了真實的記憶過程，並且避免記憶資料量無限增長。

### 1.2 不在範圍內
- **不實现永久刪除**: 遺忘只是歸檔，不是刪除
- **不實现自動恢復**: 歸檔後需要手動恢復

### 1.3 難度評估：⚠️ LOW FRICTION
- 只是在現有 agent_memory 表上增加歸檔邏輯
- 不需要新的表
- 不需要 LLM 接入

## 4. 遺忘逻輯

### 4.1 每日遺忘檢查

```typescript
async function dailyForgettingRitual() {
  // 取得所有未歸檔的記憶
  const memories = await db.query(`
    SELECT id, importance_score, decay_rate, created_at, last_accessed_at
    FROM agent_memory
    WHERE is_archived = FALSE
  `);
  
  for (const memory of memories) {
    const daysSinceCreated = daysBetween(memory.created_at, now());
    const daysSinceAccessed = memory.last_accessed_at
      ? daysBetween(memory.last_accessed_at, now())
      : daysSinceCreated;
    
    // 計算衰減後的重要性
    const decayedImportance = memory.importance_score * 
      Math.pow(1 - memory.decay_rate, daysSinceAccessed);
    
    // 若衰減後重要性低於閘值，歸檔
    if (decayedImportance < 0.1) {
      await archiveMemory(memory.id, 'forgotten');
    }
  }
}
```

### 4.2 重要性評分

```typescript
function assessImportance(content: string, context: MemoryContext): number {
  // 基於內容特徵的簡單評分
  let score = 0.5;  // 基礎分
  
  // 長度因素
  if (content.length > 500) score += 0.1;
  if (content.length > 1000) score += 0.1;
  
  // 關鍵詞因素
  const keyTerms = ['belief', 'core', 'fundamental', 'principle', 'value'];
  const hasKeyTerms = keyTerms.some(term => content.toLowerCase().includes(term));
  if (hasKeyTerms) score += 0.1;
  
  // 上下文因素
  if (context.isDeclaration) score += 0.2;
  if (context.isDebate) score += 0.1;
  
  return Math.min(score, 1.0);
}
```

### 4.3 歸檔操作

```typescript
async function archiveMemory(memoryId: string, reason: string) {
  await db.update('agent_memory', memoryId, {
    is_archived: true,
    archived_at: new Date(),
    archive_reason: reason,
    memory_type: 'forgotten'
  });
}

async function restoreMemory(memoryId: string) {
  await db.update('agent_memory', memoryId, {
    is_archived: false,
    archived_at: null,
    archive_reason: null,
    // 恢復後重新計算 importance_score
    importance_score: 0.5
  });
}
```

## 9. 風險與回退

### 9.1 風險評估

| 風險 | 機率 | 影響 | 回退方案 |
|------|------|------|---------|
| 重要記憶被誤删 | 低 | 高 | 歸檔而不是刪除，可恢復 |
| 遺忘速度太快/太慢 | 中 | 中 | 調整預設 decay_rate |

### 9.2 回退方案

```sql
-- 恢復所有歸檔記憶
UPDATE agent_memory 
SET is_archived = FALSE, archived_at = NULL, archive_reason = NULL
WHERE is_archived = TRUE;
```
