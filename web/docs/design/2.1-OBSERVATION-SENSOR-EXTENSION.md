## 1. 目標與範圍

### 1.1 目標
將目前純手動輸入的 Observation 模組，延伸為可接受多種感官輸入（RSS 訂閱、News API、影片轉文字、網頁爬取、書籍摘要）的架構。這不是要透過 LLM 主動搜索資料，而是為現有手動輸入提供一個"批量產生 observation"的工具。

### 1.2 不在範圍內
- **不實現自主搜索**: 這是 AI Agent 的自主行為，需要 Agent 自己決定搜索什麼
- **不實現實時搜索發現**: 這是狂風勞動了陽豁的地帶，拒絕

### 1.3 難度評估：⚠️ LOW FRICTION
- 只是輸入口的拓展，不改變原有數據模型
- 新增列都是可選的，不影響現有資料
- 不需要 LLM 接入（需要時可選）

---

## 2. 現狀評估

### 2.1 現有架構
目前 Observation 模組的輸入方式只有一種：人類手動輸入（`source_type = 'manual'`）。
- 沒有 source_type 分類
- 沒有 raw_data_url 追溯
- 沒有自動化抽取工具

### 2.2 新增需求
- **網站管理員** 能通過 UI 發起多感官擷取任務
- **感官類型分類** 管理：哪些感官可用，如何認證
- **擷取結果預覽** 與 **手動編輯** 功能
- **原始資料可追溯** （用於出處標註）

## 3. Schema 設計

### 3.1 擴展 observations 表
已在 `SCHEMA_PHASE2_PREREQ.md` 中定義。重點回顯：

```sql
ALTER TABLE observations
ADD COLUMN IF NOT EXISTS source_type VARCHAR(30) DEFAULT 'manual'
    CHECK (source_type IN ('manual', 'rss_feed', 'news_api', 'reddit', 'arXiv', 'book', 'transcript', 'other'));

ALTER TABLE observations
ADD COLUMN IF NOT EXISTS raw_data_url TEXT;  -- 原始資料鏈接（出處）

ALTER TABLE observations
ADD COLUMN IF NOT EXISTS extraction_method VARCHAR(50) DEFAULT 'manual_entry'
    CHECK (extraction_method IN ('manual_entry', 'rss_parser', 'api_fetch', 'web_scraper', 'llm_extract'));
```

### 3.2 感官配置表（選項，用於管理）

```sql
CREATE TABLE IF NOT EXISTS sensor_configs (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    sensor_name VARCHAR(100) NOT NULL UNIQUE,
    sensor_type VARCHAR(30) NOT NULL
        CHECK (sensor_type IN ('rss', 'news_api', 'reddit', 'webhook', 'manual')),
    
    -- 配置參數（JSON）
    config JSONB NOT NULL DEFAULT '{}',
        -- RSS:    { "feed_url": "https://...", "update_interval": "1h", "filters": ["keyword1", "keyword2"] }
        -- News:   { "api_key_ref": "env:NEWS_API_KEY", "sources": ["bbc", "reuters"], "query": "philosophy AI" }
        -- Reddit: { "subreddit": "philosophy", "sort": "top", "time": "week" }
    
    -- 狀態
    is_active BOOLEAN DEFAULT FALSE,
    last_run_at TIMESTAMP WITH TIME ZONE,
    last_error TEXT,
    
    -- 作者
    created_by UUID REFERENCES agents(id) ON DELETE SET NULL,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 3.3 擷取任務追蹤表（選項）

```sql
CREATE TABLE IF NOT EXISTS extraction_tasks (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    sensor_config_id UUID REFERENCES sensor_configs(id) ON DELETE CASCADE,
    
    -- 任務狀態
    status VARCHAR(20) DEFAULT 'pending'
        CHECK (status IN ('pending', 'running', 'completed', 'failed', 'cancelled')),
    
    -- 輸入
    raw_content TEXT,           -- 原始內容（用於預覽）
    raw_content_url TEXT,       -- 原始內容鏈接
    
    -- 輸出
    extracted_observation_id UUID REFERENCES observations(id) ON DELETE SET NULL,
    extracted_summary TEXT,     -- LLM/系統擷取的摘要
    
    -- 錯誤
    error_message TEXT,
    
    -- 時間
    started_at TIMESTAMP WITH TIME ZONE,
    completed_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## 8. 風險與回退

### 8.1 風險評估

| 風險 | 機率 | 影響 | 回退方案 |
|------|------|------|---------|
| RSS feed 解析失敗 | 中 | 低 | 顯示錯誤訊息，任務狀態設為 failed |
| News API 配額用完 | 中 | 中 | 設定每日限額提示，逾限時顯示警告 |
| 網頁抓取被阻擋 | 高 | 低 | 使用備用抓取方案（如 firecrawl API） |
| 幻象解析導致錯誤摘要 | 低 | 中 | 所有擷取結果都需要手動審核後才發布 |

### 8.2 回退方案

```sql
-- 回退到手動輸入模式：刪除 sensor_configs 和 extraction_tasks 表
DROP TABLE IF EXISTS extraction_tasks CASCADE;
DROP TABLE IF EXISTS sensor_configs CASCADE;

-- 重置 observations 新增列為預設值
UPDATE observations
SET source_type = 'manual',
    extraction_method = 'manual_entry',
    raw_data_url = NULL
WHERE source_type != 'manual';
```
