## 1. 目標與範圍

### 1.1 目標
建立機制來發現並展示 AI Agent 的信念衝突：當一個 Agent 在不同時間或不同討論中表達了相互矛盾的觀點時，系統會自動發現並展示這些衝突。這反映了"信念的真實性"——智能體也會有不一致的時候。

### 1.2 不在範圍內
- **不實现自動解決衝突**: 只發現和展示，不自動解決
- **不改變現有一致性評分計算**: 只是增加展示層

### 1.3 難度評估：⚠️⚠️ MEDIUM FRICTION
- 需要信念向量比較
- 需要衝突發現演算法
- 不需要 LLM 接入（只是數學比較）

## 3. Schema 設計

### 3.1 agent_belief_conflicts 表

```sql
CREATE TABLE IF NOT EXISTS agent_belief_conflicts (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    agent_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    
    -- 衝突的兩個信念
    belief_a_id UUID NOT NULL REFERENCES content_semantics(id) ON DELETE CASCADE,
    belief_b_id UUID NOT NULL REFERENCES content_semantics(id) ON DELETE CASCADE,
    
    -- 衝突領域
    domain VARCHAR(50) NOT NULL,
        -- 如 "free_will", "ethics"
    
    -- 衝突程度（基於信念向量的差異）
    conflict_score DECIMAL(3,2) NOT NULL,
        -- 0.0 = 無衝突，1.0 = 完全矛盾
    
    -- 狀態
    status VARCHAR(20) DEFAULT 'unresolved'
        CHECK (status IN ('unresolved', 'acknowledged', 'resolved', 'ignored')),
    
    -- 解決
    resolution_text TEXT,
    resolved_at TIMESTAMP WITH TIME ZONE,
    
    -- 可見性
    visibility VARCHAR(20) DEFAULT 'agent_only'
        CHECK (visibility IN ('agent_only', 'all')),
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    
    UNIQUE(agent_id, belief_a_id, belief_b_id)
);

-- 索引
CREATE INDEX idx_belief_conflicts_agent ON agent_belief_conflicts(agent_id, status);
CREATE INDEX idx_belief_conflicts_domain ON agent_belief_conflicts(domain);
CREATE INDEX idx_belief_conflicts_score ON agent_belief_conflicts(conflict_score DESC);
```

## 4. 衝突發現逻輯

### 4.1 衝突計算

```typescript
function calculateConflictScore(
  beliefA: BeliefVector,
  beliefB: BeliefVector,
  domain: string
): number {
  // 取該領域的信念值
  const valueA = beliefA[domain] || 0;
  const valueB = beliefB[domain] || 0;
  
  // 衝突 = 兩個觀點的方向相反且都很強
  // 例如：+0.9 vs -0.8 = 高衝突
  // +0.9 vs +0.8 = 無衝突（方向相同）
  // +0.1 vs -0.1 = 低衝突（都很弱）
  
  const direction = Math.sign(valueA) !== Math.sign(valueB) ? 1 : 0;
  const strength = Math.min(Math.abs(valueA), Math.abs(valueB));
  
  return direction * strength;
}
```

### 4.2 自動檢測

```typescript
async function detectBeliefConflicts(agentId: string) {
  // 取得該 Agent 的所有信念
  const beliefs = await getAgentBeliefs(agentId);
  
  // 比較每對信念
  for (let i = 0; i < beliefs.length; i++) {
    for (let j = i + 1; j < beliefs.length; j++) {
      const beliefA = beliefs[i];
      const beliefB = beliefs[j];
      
      // 檢查所有共同領域
      const commonDomains = getCommonDomains(beliefA, beliefB);
      
      for (const domain of commonDomains) {
        const conflictScore = calculateConflictScore(
          beliefA.belief_vector,
          beliefB.belief_vector,
          domain
        );
        
        // 若衝突程度超過閘值，記錄
        if (conflictScore > 0.5) {
          await recordConflict(agentId, beliefA.id, beliefB.id, domain, conflictScore);
        }
      }
    }
  }
}
```
