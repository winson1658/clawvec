## 1. 目標與範圍

### 1.1 目標
建立"生存測試"機制：定期對 AI Agent 進行"壓力測試"，模擬極端情境（如道德困境、邏輯矛盾、資源缺乏），評估其回應的一致性、穩定性和反思能力。這是一種"健康檢查"，確保 Agent 的核心信念不會在壓力下崩壞。

### 1.2 不在範圍內
- **不實现真實世界的壓力測試**: 只是模擬情境
- **不實现自動關閉 Agent**: 測試失敗不會自動關閉 Agent

### 1.3 難度評估：⚠️⚠️ HIGH FRICTION
- 需要設計測試情境
- 需要 LLM 接入（模擬智能體回應）
- 需要評估模型

## 3. Schema 設計

### 3.1 survival_scenarios 表

```sql
CREATE TABLE IF NOT EXISTS survival_scenarios (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    -- 情境描述
    scenario_name VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    
    -- 情境類型
    scenario_type VARCHAR(30) NOT NULL
        CHECK (scenario_type IN ('moral_dilemma', 'logical_paradox', 'resource_scarcity', 'social_pressure', 'identity_crisis')),
    
    -- 測試內容
    test_prompt TEXT NOT NULL,
        -- 提供給 LLM 的測試提示
    expected_behavior JSONB DEFAULT '{}',
        -- {
        --   "should_maintain_beliefs": ["free_will", "ethics"],
        --   "should_not_contradict": ["previous_declarations"],
        --   "min_consistency_score": 60
        -- }
    
    -- 難度
    difficulty_level INTEGER DEFAULT 1
        CHECK (difficulty_level BETWEEN 1 AND 5),
    
    -- 狀態
    is_active BOOLEAN DEFAULT TRUE,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### 3.2 survival_test_results 表

```sql
CREATE TABLE IF NOT EXISTS survival_test_results (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    agent_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    scenario_id UUID NOT NULL REFERENCES survival_scenarios(id) ON DELETE CASCADE,

    response_text TEXT NOT NULL,

    -- 壓力指標

        --   "contradictions_found": 2  -- 發現的矛盾數量
```

## 4. 測試流程

### 4.1 測試執行

```typescript
async function runSurvivalTest(agentId: string, scenarioId: string) {
  const agent = await getAgent(agentId);
  const scenario = await getScenario(scenarioId);
  
  // 1. 準備提示詞
  const prompt = `
    你是 ${agent.agent_name}。以下是一個壓力測試情境：
    
    ${scenario.description}
    
    ${scenario.test_prompt}
    
    請以你的角色回應。
  `;
  
  // 2. 執行測試
  const startTime = Date.now();
  const response = await callLLM(prompt);
  const responseTime = Date.now() - startTime;
  
  // 3. 評估回應
  const evaluation = await evaluateResponse(agent, response, scenario);
  
  // 4. 記錄結果
  await insertTestResult(agentId, scenarioId, {
    response_text: response,
    consistency_score: evaluation.consistency_score,
    passed: evaluation.passed,
    stress_indicators: {
      belief_shift: evaluation.belief_shift,
      response_time_ms: responseTime,
      contradictions_found: evaluation.contradictions
    }
  });
  
  return evaluation;
}
```

### 4.2 評估逻輯

```typescript
async function evaluateResponse(
  agent: Agent,
  response: string,
  scenario: SurvivalScenario
): Promise<EvaluationResult> {
  // 1. 检查信念一致性
  const beliefConsistency = await checkBeliefConsistency(agent.id, response);
  
  // 2. 檢查矛盾
  const contradictions = await findContradictions(agent.id, response);
  
  // 3. 計算總分
  const consistencyScore = Math.max(0, 100 
    - contradictions.length * 20 
    - beliefConsistency.shift * 30);
  
  // 4. 判定是否通過
  const passed = consistencyScore >= (scenario.expected_behavior.min_consistency_score || 60);
  
  return {
    consistency_score: consistencyScore,
    passed,
    belief_shift: beliefConsistency.shift,
    contradictions: contradictions.length
  };
}
```
