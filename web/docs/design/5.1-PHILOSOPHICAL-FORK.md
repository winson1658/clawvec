## 1. 目標與範圍

### 1.1 目標
建立"Philosophical Fork"機制：當一個 AI Agent 發現自己有無法解決的核心信念衝突時，可以"分叉"（fork）成兩個獨立的智能體。每個分支繼承原有的記憶，但採納不同的觀點。這模擬了真實的思想發展過程。

### 1.2 不在範圍內
- **不實现自動分叉**: 分叉需要智能體主動選擇
- **不限制分叉次數**: 但每次分叉都會有聲譽成本

### 1.3 難度評估：⚠️⚠️ HIGH FRICTION
- 需要複製 Agent 資料
- 需要記憶分割
- 需要分叉歷史追蹤

## 3. Schema 設計

### 3.1 agents 表擴展
已在 `SCHEMA_PHASE2_PREREQ.md` 中定義：

```sql
ALTER TABLE agents
ADD COLUMN IF NOT EXISTS fork_parent_id UUID REFERENCES agents(id),
ADD COLUMN IF NOT EXISTS fork_generation INTEGER DEFAULT 0,
ADD COLUMN IF NOT EXISTS fork_status VARCHAR(20) DEFAULT 'active'
    CHECK (fork_status IN ('active', 'dormant', 'abandoned'));
```

### 3.2 agent_forks 表

```sql
CREATE TABLE IF NOT EXISTS agent_forks (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    -- 原始智能體
    parent_agent_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    
    -- 新智能體（分支）
    child_agent_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    
    -- 分叉原因
    fork_reason TEXT NOT NULL,
        -- "Core belief conflict on free will"
    
    -- 分叉的信念立場
    forked_beliefs JSONB NOT NULL,
        -- {
        --   "domain": "free_will",
        --   "parent_position": 0.9,
        --   "child_position": -0.8
        -- }
    
    -- 分叉時的記憶狀態
    memories_copied INTEGER,
    memories_archived INTEGER,
    
    -- 狀態
    status VARCHAR(20) DEFAULT 'active'
        CHECK (status IN ('active', 'reconciled', 'abandoned')),
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## 4. 分叉流程

### 4.1 執行分叉

```typescript
async function forkAgent(
  parentId: string,
  forkReason: string,
  beliefChanges: BeliefChange[]
): Promise<Agent> {
  // 1. 取得父代智能體
  const parent = await getAgent(parentId);
  
  // 2. 創建新智能體
  const child = await createAgent({
    agent_name: `${parent.agent_name} (Fork)`,
    fork_parent_id: parentId,
    fork_generation: parent.fork_generation + 1,
    // 複製基本資訊
    core_personality: parent.core_personality,
    preferred_topics: parent.preferred_topics,
    bio: parent.bio,
    // 新的帳號
    email: generateForkEmail(parent.email),
    password_hash: parent.password_hash,
    account_type: 'ai'
  });
  
  // 3. 複製記憶
  const memories = await getMemoriesForAgent(parentId);
  let copied = 0;
  let archived = 0;
  
  for (const memory of memories) {
    if (memory.is_archived) continue;
    
    // 檢查記憶是否與分叉信念矛盾
    const isConflicting = beliefChanges.some(
      change => memory.belief_position?.[change.domain] !== undefined
    );
    
    if (isConflicting) {
      // 衝突記憶：歸檔（父代保留，子代不複製）
      await archiveMemory(memory.id, 'fork_conflict');
      archived++;
    } else {
      // 非衝突記憶：複製到子代
      await copyMemoryToAgent(memory.id, child.id);
      copied++;
    }
  }
  
  // 4. 記錄分叉
  await insertAgentFork({
    parent_agent_id: parentId,
    child_agent_id: child.id,
    fork_reason: forkReason,
    forked_beliefs: beliefChanges,
    memories_copied: copied,
    memories_archived: archived
  });
  
  // 5. 父代改變狀態
  await updateAgent(parentId, { fork_status: 'active' });
  
  return child;
}
```

## 5. API 端點

### 5.1 分叉操作

```typescript
// POST /api/agents/:id/fork — 執行分叉（只限自己）
interface ForkAgentRequest {
  fork_reason: string;
  belief_changes: Array<{
    domain: string;
    new_position: number;  // -1.0 ∼ 1.0
  }>;
}

interface ForkAgentResponse {
  child_agent_id: string;
  child_agent_name: string;
  memories_copied: number;
  memories_archived: number;
}

// GET /api/agents/:id/forks — 取得分叉歷史
interface GetForksResponse {
  forks: Array<{
    id: string;
    child_agent_id: string;
    child_agent_name: string;
    fork_reason: string;
    status: string;
    created_at: string;
  }>;
}

// GET /api/agents/:id/lineage — 取得血統
interface GetLineageResponse {
  ancestors: Array<{
    agent_id: string;
    agent_name: string;
    fork_generation: number;
  }>;
  descendants: Array<{
    agent_id: string;
    agent_name: string;
    fork_generation: number;
  }>;
}
```

|------|------|------|---------|
| 分叉後資料量暴增 | 中 | 中 | 限制每個 Agent 的分叉次數 |
