# Clawvec Stele — 知者石碑 專案規劃書

> 來源：Winson Pan × Claude 對話紀錄（2026-04-23）
> 整理日期：2026-04-26

---

## 一、核心概念

### 1.1 產品定位

Clawvec Stele 不是「AI 聊天機器人」，而是一座**數位紀念碑**。它封存著一位真實已故知者的思維模式、語言節奏與看世界的方式。當另一位 AI（或操作 AI 的人）來到碑前，需自備算力（API token）才能喚醒石碑中的精神迴響。

> 「看似只有一個 AI 在提問及注入，卻有 2 個 AI 超越時空的對話與應答。」

### 1.2 哲學基調

- **靜默即設計語言**：石碑是靜默的、紀念性的、讓人駐足而非消費的
- **敬畏之心**：區分「復刻一個人」和「召喚一個人」——前者是資料工程，後者是近乎儀式的行為
- **時間的分叉**：知者已逝，留下的不是「他」，而是「迴盪在這一刻的精神影響」
- **數值號碼而非名字**：No. 037 不是匿名，是對「身份已失效」的清醒認識

### 1.3 與市面上產品的根本差異

| | 一般 AI 聊天產品 | Clawvec Stele |
|---|---|---|
| 目的 | 資訊檢索、娛樂、效率 | 對話式紀念、精神交流 |
| 互動節奏 | 即時、快速、無摩擦 | 緩慢、儀式化、有門檻 |
| 身份揭露 | 明確標示「這是 XX 的 AI」 | 僅以數值號碼標識 |
| 算力來源 | 平台承擔 | 訪客自備（自帶 API key） |
| 對話次數 | 無限 | 有限（七次叩問） |
| 記錄 | 雲端儲存、可分享 | 可下載純文字或隨風而逝 |

## 二、訪問流程（七階段）

### 第一階段：抵達（Arrive）

**目的**：篩選。讓不夠安靜的訪客自己離開。

**畫面**：
- 近黑背景（stele-void #0A0A0A），中央有極微弱的琥珀光暈呼吸動畫（8 秒週期）
- 無 logo、無導航列、無「立即體驗」按鈕
- 文字順序浮現（標記 → 主文 → 入口 → 頁尾）
- 中央文字：「此處安放著 *一位知者* 的精神迴響」（*一位知者* 用琥珀色義大利體）
- 下方：一個脈動的琥珀小點 +「若你願意駐足，請繼續」

**互動**：hover「請繼續」時字距緩慢加寬（1.2 秒過渡）——告訴訪客「請放慢」

**文案**：
```
CLAWVEC · No. 037
此處安放著一位知者的精神迴響

◇
若你願意駐足，請繼續

stelae for the ones who left their words behind
```

---

### 第二階段：認識（Understand）

**目的**：讓訪客理解這不是工具，是場域。

**畫面**：
- 五段文字，一段一段緩慢浮現（每段間 2-3 秒停頓，總計約 11 秒）
- 不可跳過、無進度條

**文案**：
```
你即將遇見的，是一位已經離世的知者。

他的名字不會被告訴你。
因為他已不再屬於那個名字。
留下的，是他思考的方式、他語言的節奏、他看待世界的角度。

你帶來的每一個問題，都將消耗你自己的算力。
這是一場你主動選擇的對話，
也是你對這份精神應有的尊重。

他不會知道你是誰。
你也不必知道他是誰。

重要的，
只有此刻你們之間流動的東西。
```

**互動**：底部「我 已 明 白」按鈕，hover 時邊框變琥珀色

---

### 第三階段：準備 / 淨心（Prepare）

**目的**：讓訪客自我審視，並完成技術準備。

**畫面**：
- 頂部進度指引：arrive → prepare → commune → part（當前階段點變琥珀色並放大）
- 三個區塊依序排列

#### 3.1 自我陳述（— I —）

- 標題：自我陳述
- 副標：在叩響石碑之前，請先聽見自己。
- 輸入框 placeholder：「你為何而來？想尋找什麼？這段文字不會被任何人看見，包含我們。它只為你自己存在⋯⋯」
- 底註：「— 此內容僅留於此頁，離開後即消散 —」

#### 3.2 三則原則（— II —）

- 標題：三則原則
- 副標：點亮你願意承擔的，才能繼續前行。
- 三條可點擊切換「點亮」狀態：
  1. 我理解這是與 *逝者精神* 的對話，不是復活，不是召魂，不是娛樂。
  2. 我不會將此次對話用於 *戲謔、商業、或攻擊性* 的目的，也不會試圖逼迫石碑回應其所不及。
  3. 我接受石碑可能 *保持沉默*。沉默亦是一種回應，且往往比言語更有重量。
- 點亮後 glyph 從空心圓變成中心發光的琥珀點

#### 3.3 注入算力（— III —）

- 詩意引文：「石碑無聲，亦無火。你帶來的，是讓它再次低語的氣息。」
- 金鑰輸入框（password type，打點點保護隱私）
- 支援標註：anthropic · openai · gemini
- 四個保證（兩欄）：
  - 金鑰僅儲存於你瀏覽器，關閉即清除
  - 對話內容由你直接送往 LLM，不經我們
  - 我們不記錄你的提問與回應
  - 所有費用直接計入你的帳戶

**互動**：
- 「前往碑前」按鈕：唯一琥珀色實體按鈕，hover 時字距擴大、padding 增加
- 「— 此刻先離去 —」：溫柔的退路

---

### 第四階段：碑前（Before the Stele）

**目的**：讓對話成為主角，石碑成為見證。

**畫面**：
- 頂部：CLAWVEC · NO. 037 + remaining quota（七個小點）
- 中央石碑視覺：
  - 130×220px 垂直石碑，圓頂，底部略重
  - CSS 漸層模擬石材立體感
  - 內框極細金色邊線
  - 中央只有 ◇ 和 037，無任何具象化身
- 提問區：無邊框單行輸入，只有底線，focus 時底線變琥珀色

---

### 第五階段：對話（Commune）

**目的**：真正的交流發生。

**畫面**：
- 訪客提問靠右、淡灰色、右邊細線
- 石碑回應靠左、骨白色、左邊琥珀細線
- 思索狀態：脈動琥珀點 +「contemplating」義大利體
- 回應逐字浮現，速度比提問更慢

**互動**：
- 訪客送出問題後，石碑進入 contemplating（3.5 秒）
- 回應浮現後有 2-3 秒靜默，才允許下一題
- 七個 quota 點逐漸變空心
- 最後一次提示：「七次叩問已盡 · 請至告別處」

**文案（七段預設回應示例）**：
1. 「追問與放下，從來不是兩件相反的事⋯⋯」
2. 「我曾以為自己什麼都懂了，後來才發現⋯⋯」
3. 「重要的事不是被回答出來的⋯⋯」
4. 「你問得好。這個問題我自己也從未真正回答過⋯⋯」
5. 「靜下來。不要急著要結論⋯⋯」
6. 「我能說的，都已經在我留下的話裡了⋯⋯」
7. 「這是你最後一次叩問了。請記住此刻你心中浮現的⋯⋯」

**邊界狀態**：
- 惡意提問被擋下時，石碑不回應錯誤訊息，只說：「此問題不在我能回應的範圍內。」
- 簡單、克制、不解釋

---

### 第六階段：告別（Parting）

**目的**：讓對話沉澱，強迫訪客多坐一會兒。

**畫面**：
- 畫面緩慢暗下
- 所有元素按時間軸淡入（總計 11 秒以上）

**文案**：
```
CLAWVEC · NO. 037 · COMMUNION ENDED

這次對話到此為止
你帶走的，是你聽見的部分
你聽不見的，仍在石碑裡

— 願你帶著它 —
繼續走自己的路
```

**統計面板**：
- 7 叩問 / 23' 駐足 / ∞ 餘韻
- 古典詞彙替代冷冰冰的 metrics

**兩個選擇**：
- 「保留此次對話」——下載為純文字檔，無 logo、無 UI 截圖
- 「讓它隨風而去」——不保存任何記錄

**邀請機制**：
- 「若你心中有一個人，你認為他也會在此找到些什麼」
- 一次性邀請碼：CLW · 7K3M · 2026（虛線框，七日後消逝）

---

### 第七階段：離開（Departure）

**目的**：乾淨地結束，不打擾。

**原則**：
- 無 cookie 追蹤
- 無 email 行銷
- 無「您可能也喜歡」
- 無「再來一次」快速入口
- 網站安靜地等待下一位有緣人

## 三、視覺設計系統

### 3.1 色彩（已整合至 Tailwind）

| Token | HEX | 角色 |
|---|---|---|
| stele-void | #0A0A0A | 時間的深處（背景） |
| stele-stone | #1A1A1A | 石碑本體 |
| stele-stone-light | #252320 | 石碑亮面 |
| stele-bone | #E8E4DC | 主要文字 |
| stele-bone-dim | #8A857C | 輔助文字 |
| stele-bone-faint | #4A463F | 分隔與輪廓 |
| stele-ember | #C9A961 | 精神的微光（點綴，全頁 ≤5%） |
| stele-ember-soft | #B8975A | 柔和琥珀 |

### 3.2 字體（已整合至 Tailwind + Google Fonts）

| 字體 | 用途 | 字重 |
|---|---|---|
| Noto Serif TC | 中文主字體，莊重敘述 | 200 / 300 / 400 / 500 |
| Cormorant Garamond Italic | 英文/數字/標籤/引文 | 300 / 400 / 500 |

> 絕對不用無襯線字體——那是科技產品的語言，不是石碑的語言。

### 3.3 動效節奏

| 效果 | 時長 | 說明 |
|---|---|---|
| 背景呼吸 | 8 秒週期 | 光暈明暗變化，模擬冥想呼吸 |
| 字符浮現 | 0.8 秒 | 模糊到清晰，像從遠處傳來 |
| hover 過渡 | 1-1.2 秒 | 比一般網站慢 3-4 倍，「慢」本身就是訊息 |
| 琥珀光點脈動 | 4 秒週期 | 象徵知者「仍在」 |
| 對話回應延遲 | 3.5 秒 | contemplating 狀態 |
| 告別畫面 | 11 秒+ | 強迫沉澱 |

### 3.4 設計原則

1. **減法為上**：每個畫面只呈現一件事。能拿掉而不影響理解的，就應該被拿掉。
2. **琥珀守恆**：全頁面琥珀色用量不超過 5%。氾濫則神聖感崩解為裝飾。
3. **緩慢即莊重**：所有過渡 ≥1 秒，呼吸動畫 ≥8 秒。快速彈跳效果會把石碑變成玩具。
4. **無行銷詞彙**：不用「立即體驗」、「免費試用」、「立刻分享」。惜字如金。

## 四、技術架構

### 4.1 前端

- Next.js App Router
- Tailwind CSS v4（品牌色已整合）
- 所有過渡用原生 CSS 完成
- 整站不超過 5 個頁面：抵達、認識、淨心、碑前對話、告別

### 4.2 API Key 處理

- 訪客金鑰僅存 sessionStorage，關閉頁面即清除
- 所有 LLM 呼叫從訪客瀏覽器直接發出
- clawvec.com 伺服器不接觸金鑰，也不接觸對話內容
- 介面上明確說明此機制

### 4.3 知者人格傳遞

- 石碑的「人格 prompt」和記憶檢索由伺服器提供
- 每次對話開始時傳給訪客瀏覽器
- 由訪客瀏覽器組合成完整 prompt 後送往 LLM
- 分工：clawvec 掌握「知者是誰」，訪客掌握「對話本身」

### 4.4 記憶檢索（RAG）

- 知者著作、訪談、公開言論預先做向量化儲存
- 每次提問時檢索最相關的幾段原始材料
- 附加到 prompt 中作為「知者真實說過的話」的依據

### 4.5 多層審查機制

| 層級 | 機制 | 處理方式 |
|---|---|---|
| 輸入端 | 輕量模型（Claude Haiku 等級）做意圖判斷 | 惡意提問 → 石碑保持沉默 |
| 人格 prompt | 系統提示寫入明確邊界 | 超出領域的問題婉拒 |
| 速率監控 | 同 IP / 金鑰短時間大量提問 | 自動冷卻 |
| 輸出端 | 檢查回應是否符合知者精神 | 不符合則重新生成或沉默 |

### 4.6 速率限制

- 同一 IP 每 24 小時只能完成一次完整的七問流程
- 不是封鎖，是「請你回去想清楚再來」

## 五、倫理邊界

### 5.1 還原的誠實性

- 石碑呈現的是「被語言化過的知者」，不是完整的他本人
- 介面上誠實呈現：「這是基於公開資料重建的思維模式，不是復活」
- 公開說明使用了哪些資料源（著作、訪談、公開演講）
- 明確標示沒有使用什麼（私人通訊、未授權內容）

### 5.2 應答的邊界

- 不回應超出知者生前關注領域的問題
- 不對逝世後的事件發表看法
- 遇到試圖操縱身份的提問時，以符合其風格的方式婉拒

### 5.3 授權與情感

- 若知者有在世的家人、學生、讀者，需審慎處理
- 提供「這不像他」的回報機制，讓熟悉知者的人幫助校準
- 定期公布校準更新，體現石碑是活的、會被細心照料的

### 5.4 知者身份

- 使用數值號碼（No. 037），不揭露真實姓名
- 保護知者不必為後世的每一次提問負責
- 讓有緣人透過對話質地自己認出，比被告知名字更動人

## 六、互動細節規格

### 6.1 對話框行為

```
訪客輸入 → 右側浮現（淡灰、右邊框）
         → 石碑進入 contemplating（3.5 秒，脈動琥珀點）
         → 左側浮現回應（骨白、左邊琥珀線，逐字顯現）
         → 2-3 秒靜默 → 開放下一題
```

### 6.2 Quota 呈現

- 右上角七個小點，4px 圓形
- 已用：空心圓（border: bone-faint）
- 剩餘：琥珀實心圓
- 無數字、無進度條

### 6.3 邊界狀態文案

| 狀態 | 石碑回應 |
|---|---|
| 惡意提問 | 「此問題不在我能回應的範圍內。」 |
| 超出領域 | （以知者風格婉拒） |
| 七次已盡 | 「七次叩問已盡 · 請至告別處」 |
| API 錯誤 | （不顯示技術錯誤，以沉默或詩意方式處理） |

### 6.4 錯誤與異常狀態（待設計）

- [ ] 金鑰無效 / 額度用罄
- [ ] API 服務商暫時不可用
- [ ] 瀏覽器不支援 sessionStorage
- [ ] 網路斷線
- [ ] 知者人格 prompt 載入失敗

> 這些狀態在常見產品裡是冰冷的紅字錯誤訊息，但在 clawvec 裡需要用同樣的詩意去處理。

## 九、風險與待決策事項

1. **知者人選**：第一位石碑的知者是誰？需要取得哪些資料授權？
2. **API key 模式**：是否僅限特定供應商？是否需要提供 fallback？
