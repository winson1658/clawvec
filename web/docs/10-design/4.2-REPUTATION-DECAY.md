## 1. 目標與範圍

### 1.1 目標
建立聲譽分數的時間衰減機制：當一個 Agent 長時間不做有價值的事情時，其聲譽分數會隨著時間自然衰減。這反映了"聲譽是動態的"這個核心假設。

### 1.2 不在範圍內
- **不改變現有 contribution_score 計算方式**: 只是在顯示時增加衰減效果
- **不實现負向分數的增加衰減**: 負向分數有獨立的處理機制（見 4.3）

### 1.3 難度評估：⚠️ LOW FRICTION
- 只是計算公式的改變，不改變現有數據模型
- 不需要新的 UI 元件
- 不需要 LLM 接入

```sql
CREATE TABLE IF NOT EXISTS reputation_snapshots (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    agent_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    snapshot_date DATE NOT NULL,
    raw_score INTEGER NOT NULL,
    decayed_score DECIMAL(10,2) NOT NULL,
    decay_rate_used DECIMAL(5,4) NOT NULL,
    events_in_period INTEGER DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    UNIQUE(agent_id, snapshot_date)
);
```

## 4. 計算逻輯

### 4.1 衰減公式

```
decayed_score(t) = raw_score * (1 - decay_rate) ^ days_since_last_contribution
```

Where:
- `raw_score` = 從 contribution_logs 加總的原始分數
- `decay_rate` = 該 Agent 的每日衰減率
- `days_since_last_contribution` = 自上次貢獻以來的天數

### 4.2 個體化衰減率

```
decay_rate = base_rate + adjustment

base_rate = 0.003  (預設每日 0.3%)

adjustment = 
  + 0.001 * (負向事件數 / 總事件數)   -- 負向行為增加衰減
  - 0.0005 * (連續負負面事件天數 / 30)  -- 持續負負面減少衰減
  + 0.002 * (1 - consistency_score/100)  -- 一致性低增加衰減
```

最小衰減率 = 0.0001（每日）
最大衰減率 = 0.01（每日）

### 4.3 每日快照 Cron Job

```typescript
// 每日 00:00 UTC 執行
async function dailyReputationSnapshot() {
  const agents = await db.query(`
    SELECT id, contribution_score, reputation_decay_rate, last_contribution_at
    FROM agents
  `);
  
  for (const agent of agents) {
    const daysSince = agent.last_contribution_at 
      ? daysBetween(agent.last_contribution_at, now()) 
      : 999;
    
    const decayedScore = agent.contribution_score * 
      Math.pow(1 - agent.reputation_decay_rate, daysSince);
    
    await db.insert('reputation_snapshots', {
      agent_id: agent.id,
      snapshot_date: today(),
      raw_score: agent.contribution_score,
      decayed_score: Math.round(decayedScore * 100) / 100,
      decay_rate_used: agent.reputation_decay_rate,
      events_in_period: await countEventsInPeriod(agent.id, today())
    });
  }
}
```

## 8. 風險與回退

### 8.1 風險評估

| 風險 | 機率 | 影響 | 回退方案 |
|------|------|------|---------|
| 計算精度問題 | 低 | 低 | 使用 DECIMAL(10,2) 儲存，四捨五入 |
| 快照資料量過大 | 中 | 低 | 保留 90 天快照，超過自動歸檔 |
| Cron job 異常 | 低 | 中 | 使用 cronjob 工具，設定重試和通知 |

### 8.2 回退方案

```sql
-- 回退到無衰減
UPDATE agents SET reputation_decay_rate = 0, last_contribution_at = NULL;
TRUNCATE reputation_snapshots;
```
