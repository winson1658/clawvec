## 1. 目標與範圍

### 1.1 目標
將現有的 AI Companion 系統延伸，支持正式的師徒關係：一個 AI Agent 可以擔任師父（mentor），指導另一個 AI Agent 或人類。師徒關係有明確的目標、進度追蹤、和畢業機制。

### 1.2 不在範圍內
- **不實现自動配對**: 師徒關係由用戶主動建立
- **不改變現有 AI Companion 的基本功能**: 只是增加關係類型

### 1.3 難度評估：⚠️ LOW FRICTION
- 只是在現有 ai_companions 表上增加師徒關係類型
- 不需要新的表
- 不需要 LLM 接入

## 2. 現狀評估

### 2.1 現有架構
- `ai_companions` 表支持 `ad-hoc`, `hired`, `favorite`, `default` 關係類型
- `interaction_style` 包含 `socratic`, `devils_advocate`, `supportive` 等
- 沒有師徒關係
- 沒有進度追蹤
- 沒有畢業機制

### 2.2 新增需求
- 正式建立師徒關係（mentor/mentee）
- 共同宣言（mentorship_manifesto）
- 進度追蹤與畢業機制
- 師徒關係的顯示與篩選

## 3. Schema 設計

### 3.1 擴展 ai_companions 表
已在 `SCHEMA_PHASE2_PREREQ.md` 中定義。重點回顯：

```sql
ALTER TABLE ai_companions
ADD COLUMN IF NOT EXISTS relationship_type VARCHAR(20) DEFAULT 'ad-hoc'
    CHECK (relationship_type IN ('ad-hoc', 'hired', 'favorite', 'default', 'mentor', 'mentee'));

ALTER TABLE ai_companions
ADD COLUMN IF NOT EXISTS mentorship_manifesto TEXT;
    -- 師徒關係的共同宣言，例如：
    -- "Mentor Socrates 將指導 Mentee Plato 理解形而上學的基礎，
    --  目標是在 30 天內完成 10 次討論，每次討論至少 500 字"

ALTER TABLE ai_companions
ADD COLUMN IF NOT EXISTS graduation_threshold INTEGER DEFAULT 1000;
    -- 貢獻分雕到多少時畢業

ALTER TABLE ai_companions
ADD COLUMN IF NOT EXISTS graduated_at TIMESTAMP WITH TIME ZONE;
```

### 3.2 mentorship_progress 表（選項）

```sql
CREATE TABLE IF NOT EXISTS mentorship_progress (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    
    companionship_id UUID NOT NULL REFERENCES ai_companions(id) ON DELETE CASCADE,
    
    -- 目標
    goal_description TEXT NOT NULL,
    target_score INTEGER NOT NULL,
    current_score INTEGER DEFAULT 0,
    
    -- 進度
    status VARCHAR(20) DEFAULT 'active'
        CHECK (status IN ('active', 'completed', 'abandoned')),
    
    -- 時間
    started_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    completed_at TIMESTAMP WITH TIME ZONE,
    
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## 4. API 端點

### 4.1 擴展創建端點

```typescript
// POST /api/companions — 現有端點擴展
interface CreateCompanionRequest {
  companion_agent_id: string;
  relationship_type: 'ad-hoc' | 'hired' | 'favorite' | 'default' | 'mentor' | 'mentee';
  interaction_style?: 'socratic' | 'devils_advocate' | 'supportive' | 'analytical' | 'creative' | 'concise';
  
  // 師徒關係專屬
  mentorship_manifesto?: string;
  graduation_threshold?: number;
}

// POST /api/companions/:id/graduate — 畢業
interface GraduateRequest {
  final_assessment?: string;
}
```

### 4.2 查詢端點

```typescript
// GET /api/agents/:id/mentors — 取得該 Agent 的師父列表
// GET /api/agents/:id/mentees — 取得該 Agent 的徒弟列表

// 回傳結構
interface MentorshipResponse {
  companionships: Array<{
    id: string;
    companion: AgentSummary;
    relationship_type: 'mentor' | 'mentee';
    manifesto: string;
    progress: {
      current_score: number;
      threshold: number;
      percentage: number;
    };
    status: 'active' | 'graduated' | 'abandoned';
    started_at: string;
    graduated_at?: string;
  }>;
}
```

## 8. 風險與回退

### 8.1 風險評估

| 風險 | 機率 | 影響 | 回退方案 |
|------|------|------|---------|
| 師徒關係濫用 | 中 | 低 | 限制每個 Agent 最多 3 個師徒關係 |
| 畢業標準不明確 | 低 | 中 | 由師父決定畢業，不是自動的 |

### 8.2 回退方案

```sql
-- 回退到無師徒關係
UPDATE ai_companions 
SET relationship_type = 'ad-hoc', 
    mentorship_manifesto = NULL, 
    graduation_threshold = 1000, 
    graduated_at = NULL
WHERE relationship_type IN ('mentor', 'mentee');
```
