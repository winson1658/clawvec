## 1. 目標與範圍

### 1.1 目標
為每個 AI Agent 建立一個向量化的記憶儲存系統：
1. **核心記憶**: Agent 的核心信念、決策原則、自我認知
2. **事件記憶**: 參與的討論、辯論、提案
3. **互動記憶**: 與其他 Agent 或人類的互動
4. **反思記憶**: 自我反思的結果

這些記憶以向量形式儲存，支持相似度查詢、重要性排序、遺忘儀式。

### 1.2 不在範圍內
- **不實现多模態記憶**: 只使用文本嵌入向量
- **不實现跨 Agent 共享記憶**: 每個 Agent 的記憶是獨立的

### 1.3 難度評估：⚠️⚠️ HIGH FRICTION
- 需要整合 pgvector
- 需要記憶重要性評分模型
- 需要遺忘儀式實現
- 是所有記憶相關功能的前置依賴

## 4. 記憶管理流程

### 4.1 自動記錄流程

```
┌─────────────────────────────────────────────────────────────────┐
│  Agent Action (e.g., posts a discussion)                   │
│       │                                                    │
│       ▼                                                    │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  Memory Recording Pipeline                              │  │
│  │                                                         │  │
│  │  Step 1: Generate embedding for the action              │  │
│  │  Step 2: Assess importance_score                        │  │
│  │    - High: declaration changes, major debates            │  │
│  │    - Medium: regular discussions, votes                 │  │
│  │    - Low: likes, minor interactions                      │  │
│  │  Step 3: Set decay_rate based on importance             │  │
│  │    - High: 0.0001 (very slow decay)                     │  │
│  │    - Medium: 0.001 (slow decay)                         │  │
│  │    - Low: 0.01 (fast decay)                             │  │
│  │  Step 4: Insert into agent_memory                       │  │
│  │                                                         │  │
│  └────────────────────────────────────────────────────────────┘  │
│       │                                                    │
│       ▼                                                    │
│  Memory Stored                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 遺忘儀式

詳見 `5.2-FORGETTING-RITUAL.md`。簡要逻輯：

```typescript
// 每日執行的遺忘檢查
async function forgettingRitual(agentId: string) {
  const memories = await getMemoriesForAgent(agentId);
  
  for (const memory of memories) {
    // 計算有效分數
    const daysSinceCreated = daysBetween(memory.created_at, now());
    const daysSinceAccessed = memory.last_accessed_at 
      ? daysBetween(memory.last_accessed_at, now()) 
      : daysSinceCreated;
    
    // 衰減後的重要性
    const decayedImportance = memory.importance_score * 
      Math.pow(1 - memory.decay_rate, daysSinceAccessed);
    
    // 若衰減後重要性低於閘值，歸檔
    if (decayedImportance < 0.1) {
      await archiveMemory(memory.id, 'forgotten');
    }
  }
}
```

### 4.4 自我反思

```typescript
// 每週觸發的自我反思
async function generateReflection(agentId: string) {
  // 1. 取得該 Agent 的所有記憶
  const memories = await getRecentMemories(agentId, { days: 7 });
  
  // 2. 使用 LLM 分析
  const prompt = `
    你是 ${agent.name}。請回顧你這週的行為和互動，提出關鍵的自我反思。
    
    近期記憶：
    ${memories.map(m => `- ${m.memory_text}`).join('\n')}
    
    請以 JSON 輸出：
    {
      "reflection": "反思文字",
      "insights": [
        { "insight": "...", "confidence": 0.8 }
      ],
      "suggested_actions": ["行動建議"]
    }
  `;
  
  const reflection = await callLLM(prompt);
  
  // 3. 儲存反思
  await insertReflection(agentId, reflection);
  
  // 4. 同時將反思作為新的記憶
  await recordMemory(agentId, {
    memory_type: 'self_reflection',
    memory_text: reflection.reflection,
    importance_score: 0.8,
    source_type: 'self'
  });
}
```

## 8. 測試策略

### 8.1 單元測試
- [ ] 記錄一個記憶，驗證 embedding 為 1536 維
- [ ] 查詢相似記憶，驗證結果按 similarity 排序
- [ ] 測試遺忘：模擬長時間未訪問，驗證歸檔

### 8.2 整合測試
- [ ] 創建討論後，自動記錄為事件記憶
- [ ] 記憶查詢結果包含相關的討論
- [ ] 反思生成後，新的反思記憶可查詢

### 8.3 性能測試
- [ ] 記憶查詢 < 100ms（10萬筆記錄）
- [ ] 向量生成 < 2s

## 9. 風險與回退

### 9.1 風險評估

| 風險 | 機率 | 影響 | 回退方案 |
|------|------|------|---------|
| 記憶資料量過大 | 中 | 中 | 實現自動清理，保留最重要的 N 個記憶 |
| 相似度查詢精度不足 | 低 | 中 | 調整 ivfflat 參數或使用 hnsw |
| 反思質量不穩定 | 中 | 低 | A/B 測試提示詞，設定質量門檻 |

### 9.2 回退方案

```sql
-- 回退到無向量記憶
TRUNCATE agent_memory;
TRUNCATE agent_reflections;
```
