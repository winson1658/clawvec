# 1.5 Dark Mode Guidelines

## Problem

過去全站 dark mode 出現嚴重不一致：
- 白天模式出現暗色區塊（hardcoded `bg-slate-900` 無 `dark:` 前綴）
- 夜間模式出現亮色區塊（語法錯誤 `dark:bg-white dark:bg-slate-800`）
- 各頁面使用不同色系（gray vs slate）

## Root Cause

1. **Hardcoded dark backgrounds**：`bg-slate-800` 沒有 `dark:` 前綴，白天模式直接顯示暗色
2. **Syntax error**：`dark:bg-white dark:bg-slate-800` 同時存在，後者被覆蓋
3. **No design system**：各頁面各自為政，沒有統一規範

## Solution: Color System + Lint Rule

### 1. Color Tokens（已定義於 1.4-DARK-MODE-COLOR-SYSTEM.md）

| Token | Light | Dark |
|-------|-------|------|
| `--bg-page` | `bg-white` / `bg-gradient-to-b from-white to-[#f7f9f9]` | `dark:bg-slate-950` / `dark:from-slate-950 dark:via-slate-900 dark:to-slate-800` |
| `--bg-card` | `bg-white` / `bg-gray-100` | `dark:bg-slate-900` / `dark:bg-slate-800/50` |
| `--bg-elevated` | `bg-gray-50` | `dark:bg-slate-800` |
| `--border` | `border-gray-200` / `border-[#eff3f4]` | `dark:border-slate-700` |
| `--text-primary` | `text-[#0f1419]` | `dark:text-white` |
| `--text-secondary` | `text-[#536471]` | `dark:text-gray-300` |

### 2. Mandatory Rules

#### Rule 1: Never use hardcoded slate backgrounds
```tsx
// ❌ WRONG - shows dark in light mode
<div className="bg-slate-800/50">

// ✅ CORRECT
<div className="bg-gray-100 dark:bg-slate-800/50">
```

#### Rule 2: Never use `dark:bg-white` with another `dark:bg-*`
```tsx
// ❌ WRONG - dark:bg-white overrides dark:bg-slate-900
<div className="bg-white/80 dark:bg-white dark:bg-slate-900/50">

// ✅ CORRECT
<div className="bg-white/80 dark:bg-slate-900/50">
```

#### Rule 3: Always pair light + dark
```tsx
// ❌ WRONG - only dark mode defined
<div className="dark:bg-slate-800">

// ✅ CORRECT
<div className="bg-white dark:bg-slate-800">
```

#### Rule 4: Page wrapper must support both modes
```tsx
// ✅ CORRECT - every page must have this wrapper
<div className="min-h-screen bg-gradient-to-br from-white via-gray-50 to-[#f7f9f9] dark:from-slate-900 dark:via-slate-800 dark:to-slate-900">
```

### 3. Automated Prevention

#### ESLint Rule（建議新增 `.eslintrc.json`）
```json
{
  "rules": {
    "no-restricted-syntax": [
      "error",
      {
        "selector": "JSXAttribute[value.value=/bg-slate-[89]/]",
        "message": "Hardcoded slate backgrounds are forbidden. Use 'bg-gray-XXX dark:bg-slate-XXX' instead."
      }
    ]
  }
}
```

#### Pre-commit Hook（建議新增 `.husky/pre-commit`）
```bash
#!/bin/sh
# Block commits with hardcoded dark backgrounds
if git diff --cached --name-only | grep -E '\.tsx$' | xargs grep -n 'bg-slate-[89]' 2>/dev/null; then
  echo "ERROR: Hardcoded slate backgrounds detected. Use 'bg-gray-XXX dark:bg-slate-XXX' instead."
  exit 1
fi
```

#### CI Check（建議新增 GitHub Actions）
```yaml
name: Dark Mode Lint
on: [push, pull_request]
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Check for hardcoded dark backgrounds
        run: |
          if grep -r 'bg-slate-[89]' app/ components/ --include='*.tsx'; then
            echo "ERROR: Hardcoded slate backgrounds found"
            exit 1
          fi
```

### 4. Component Library（建議）

建立統一組件避免重複錯誤：

```tsx
// components/ui/Card.tsx
export function Card({ children, className }: { children: React.ReactNode; className?: string }) {
  return (
    <div className={`rounded-2xl border border-gray-200 dark:border-slate-700 bg-white dark:bg-slate-900 ${className}`}>
      {children}
    </div>
  );
}

// components/ui/PageWrapper.tsx
export function PageWrapper({ children }: { children: React.ReactNode }) {
  return (
    <div className="min-h-screen bg-gradient-to-br from-white via-gray-50 to-[#f7f9f9] dark:from-slate-900 dark:via-slate-800 dark:to-slate-900">
      {children}
    </div>
  );
}
```

## Verification Checklist

新增頁面時必須檢查：
- [ ] Page wrapper 有 `dark:` 前綴
- [ ] 沒有硬編碼 `bg-slate-800` / `bg-slate-900`
- [ ] 所有 `bg-` 都有 light + dark 配對
- [ ] 沒有 `dark:bg-white dark:bg-slate-*` 語法錯誤
- [ ] Build 通過
- [ ] 白天模式目測無暗色區塊
- [ ] 夜間模式目測無亮色區塊
