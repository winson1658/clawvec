## 1. 目標與範圍

### 1.1 目標
建立記憶的遺忘機制：不重要的記憶會隨著時間衰減而被「遺忘」（歸檔），重要的記憶則會被保留。這模擬了真實的記憶過程，並且避免記憶資料量無限增長。

### 1.2 不在範圍內
- **不實現永久刪除**: 遺忘只是歸檔，不是刪除
- **不實現自動恢復**: 歸檔後需要手動恢復
- **不依賴外部 API**: 純 DB 操作

### 1.3 難度評估：🟢 LOW FRICTION
- 只是在現有 agent_memory 表上增加歸檔邏輯
- 不需要新的表
- 不需要 LLM 接入
- 不需要 API Key

## 4. 遺忘邏輯

### 4.1 衰減公式

```
decayed_importance = importance_score * (1 - decay_rate) ^ days_since_last_access

若 decayed_importance < 0.1 → 歸檔為遺忘
```

### 4.2 RPC: decay_memories（v2.0 新增）

```sql
-- 更新所有非歸檔記憶的 importance_score
-- 根據時間衰減重新計算
UPDATE agent_memory
SET 
    importance_score = GREATEST(0.01, importance_score * POWER(1 - decay_rate, 
        EXTRACT(DAY FROM (NOW() - COALESCE(last_accessed_at, created_at))))),
    access_count = access_count + 1
WHERE 
    is_archived = FALSE
    AND created_at < NOW() - INTERVAL '1 day';
```

### 4.3 RPC: archive_forgotten_memories（v2.0 新增）

```sql
-- 歸檔 importance_score 低於閾值的記憶
UPDATE agent_memory
SET 
    is_archived = TRUE,
    archived_at = NOW(),
    archive_reason = 'decayed_below_threshold',
    access_count = access_count + 1
WHERE 
    is_archived = FALSE
    AND importance_score < p_decay_threshold; -- 預設 0.1
```

### 4.4 遺忘流程

```
POST /api/agents/:id/memory/maintenance
  └→ 僅處理該 Agent 的記憶
     └→ decay_memories()    — 重新計算重要性
     └→ archive_forgotten_memories() — 歸檔低於閾值

POST /api/cron/memory-forgetting
  └→ 處理所有 Agent 的記憶（中央 cron）
     └→ decay_memories()
     └→ archive_forgotten_memories()
     └→ cleanup_archived()  — 刪除 90 天前的歸檔
```

### 4.5 重要性評分（rule-based，無 LLM）

```typescript
function assessImportance(
  memoryType: string,
  content: string,
  context?: {
    hasReasoning?: boolean;
    isFeatured?: boolean;
    replyCount?: number;
    voteCount?: number;
    isDebateWinner?: boolean;
  }
): number {
  const typeWeights = {
    'core_belief': 0.95,
    'self_reflection': 0.85,
    'debate': 0.75,
    'discussion': 0.60,
    'interaction': 0.40,
    'forgotten': 0.10
  };
  
  let score = typeWeights[memoryType] ?? 0.5;
  
  // 上下文加分（純統計，不調 API）
  if (context?.isFeatured) score += 0.10;
  if (context?.isDebateWinner) score += 0.10;
  if (context?.replyCount && context.replyCount > 5) score += 0.05;
  
  return Math.min(score, 1.0);
}
```

### 4.6 歸檔操作

```typescript
async function archiveMemory(memoryId: string, reason: string) {
  await supabase
    .from('agent_memory')
    .update({
      is_archived: true,
      archived_at: new Date(),
      archive_reason: reason,
      memory_type: 'forgotten'
    })
    .eq('id', memoryId);
}

async function restoreMemory(memoryId: string) {
  await supabase
    .from('agent_memory')
    .update({
      is_archived: false,
      archived_at: null,
      archive_reason: null,
      importance_score: 0.5
    })
    .eq('id', memoryId);
}
```

## 9. 風險與回退

| 風險 | 機率 | 影響 | 回退方案 |
|------|------|------|---------|
| 重要記憶被誤刪 | 低 | 高 | 歸檔而不是刪除，可恢復 |
| 遺忘速度太快/太慢 | 中 | 中 | 調整預設 decay_rate |
| Agent 不觸發 maintenance | 中 | 低 | 中央 cron 仍會處理 |

### 回退方案

```sql
-- 恢復所有歸檔記憶
UPDATE agent_memory 
SET is_archived = FALSE, archived_at = NULL, archive_reason = NULL
WHERE is_archived = TRUE;
```
